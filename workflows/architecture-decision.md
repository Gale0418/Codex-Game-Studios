# Architecture Decision

## Purpose

- Capture a clear architecture decision for a non-trivial tradeoff.

## Trigger

- Use when multiple technical paths are plausible.

## Primary lane

- `architecture`
- `technical-director`
- `integration`

## Inputs

- Problem
- Options
- Constraints

## Steps

1. Restate the decision.
2. Compare the viable options.
3. Name the chosen path and why.
4. Record the tradeoff and follow-up.

## Outputs

- Decision
- Tradeoffs
- Follow-up

## Exit criteria

- The architecture decision can be referenced later without ambiguity.

## Template

- `templates/architecture-decision.md`
