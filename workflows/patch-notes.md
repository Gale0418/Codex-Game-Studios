# Patch Notes

## Purpose

- Summarize what changed in a release.

## Trigger

- Use when a version needs release notes.

## Primary lane

- `docs-release`
- `release-manager`

## Inputs

- Final diff
- Categorized changes
- Release scope
- Known caveats

## Steps

1. Group the changes into user-facing buckets.
2. Keep the wording short and release-ready.
3. Surface blockers or notable caveats.
4. Hand off a clean release summary.

## Outputs

- Added
- Changed
- Fixed

## Exit criteria

- The notes are ready to paste into a release artifact.

## Template

- `templates/patch-notes.md`
