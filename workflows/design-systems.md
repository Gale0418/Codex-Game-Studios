# Design Systems

## Purpose

- Shape the game systems, rules, and boundaries before implementation.

## Trigger

- Use when the pluralized front-door command is used for systems shaping.

## Primary lane

- `architecture`
- `game-rules`
- `ui-ux`

## Inputs

- Concept
- Source of truth files
- Rule constraints

## Steps

1. Break the concept into systems.
2. Define the source of truth for each system.
3. Call out boundaries and dependencies.
4. Hand back the design outline.

## Outputs

- Systems
- Boundaries
- Notes

## Exit criteria

- The design can be routed into implementation without ambiguity.

## Template

- `templates/design-system-outline.md`
