# Architecture Review

## Purpose

- Review a proposed architecture for risk, fit, and completeness.

## Trigger

- Use when a plan needs a technical review before implementation.

## Primary lane

- `architecture`
- `technical-director`
- `qa-regression`

## Inputs

- Architecture notes
- Constraints
- Source of truth

## Steps

1. Check the boundaries and ownership.
2. Call out missing pieces or risky assumptions.
3. Confirm whether the plan fits the problem.
4. Hand back the review result.

## Outputs

- Findings
- Risks
- Recommendation

## Exit criteria

- The architecture is either approved or explicitly blocked.

## Template

- `templates/architecture-review.md`
