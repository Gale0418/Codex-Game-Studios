# Asset Spec

## Purpose

- Define the assets, formats, and constraints needed for a feature.

## Trigger

- Use when assets need explicit requirements before production.

## Primary lane

- `art-director`
- `technical-artist`
- `production`

## Inputs

- Asset list
- Usage constraints
- Platform limits

## Steps

1. List the required assets.
2. Note format and size constraints.
3. Call out ownership and dependencies.
4. Hand back the asset specification.

## Outputs

- Asset list
- Constraints
- Notes

## Exit criteria

- The asset work can start without ambiguity.

## Template

- `templates/asset-spec.md`
