# Create Stories

## Purpose

- Convert epics into implementation-ready stories.

## Trigger

- Use when epic-level planning needs story-level detail.

## Primary lane

- `producer`
- `implementation`
- `qa-regression`

## Inputs

- Epic list
- Acceptance expectations
- Dependencies

## Steps

1. Break each epic into story-sized work.
2. Add acceptance notes for each story.
3. Order the stories by dependency.
4. Hand back the story plan.

## Outputs

- Stories
- Acceptance notes
- Ordering

## Exit criteria

- The next implementation slice is unambiguous.

## Template

- `templates/create-stories-plan.md`
