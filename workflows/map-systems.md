# Map Systems

## Purpose

- Turn a concept into systems, dependencies, and priority order.

## Trigger

- Use when a feature or game idea needs decomposition.

## Primary lane

- `architecture`
- `game-rules`
- `persistence`
- `ui-ux`

## Inputs

- Concept notes
- Source of truth files
- Current architecture notes
- Risk constraints

## Steps

1. Break the concept into systems and shared dependencies.
2. Identify what must be centralized before implementation.
3. Order the work from highest-risk to lowest-risk.
4. Hand off the map plus the recommended split.

## Outputs

- System map
- Dependency graph
- Priority order

## Exit criteria

- The feature can be split into lanes without guessing.

- `templates/system-map.md`
