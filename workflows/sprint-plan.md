# Sprint Plan

## Purpose

- Turn a scope into a short execution plan.

## Trigger

- Use when a larger body of work needs sequencing.

## Primary lane

- `producer`
- `architecture`
- `implementation`

## Inputs

- Scope
- Constraints
- Dependencies
- Target milestones

## Steps

1. Break the scope into work items.
2. Order them by dependency and risk.
3. Assign lane ownership or handoff points.
4. Leave a short verification plan for each item.

## Outputs

- Goals
- Work items
- Dependencies

## Exit criteria

- The next increment can be started without re-planning.

## Template

- `templates/sprint-plan.md`
