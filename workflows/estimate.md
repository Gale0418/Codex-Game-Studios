# Estimate

## Purpose

- Estimate scope, effort, or complexity for a task.

## Trigger

- Use when planning needs a rough but explicit estimate.

## Primary lane

- `producer`
- `architecture`
- `implementation`

## Inputs

- Scope
- Constraints
- Similar work

## Steps

1. Restate the scope.
2. Estimate the likely effort.
3. Note the biggest uncertainty.
4. Hand back the estimate and risk.

## Outputs

- Scope
- Estimate
- Risks

## Exit criteria

- The estimate is good enough to plan with.

- `templates/estimate.md`
