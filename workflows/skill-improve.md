# Skill Improve

## Purpose

- Improve a skill or workflow based on test results.

## Trigger

- Use when a skill needs refinement after testing.

## Primary lane

- `docs-release`
- `integration`
- `qa-regression`

## Inputs

- Skill
- Test feedback
- Improvement target

## Steps

1. Identify the weakest point.
2. Apply the smallest useful improvement.
3. Recheck the contract shape.
4. Hand back the improvement note.

## Outputs

- Skill
- Improvement
- Validation

## Exit criteria

- The skill is measurably better or unchanged by choice.

## Template

- `templates/skill-improve.md`
