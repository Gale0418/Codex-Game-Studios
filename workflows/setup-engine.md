# Setup Engine

## Purpose

- Establish the baseline engine, stack, or project setup needed before work starts.

## Trigger

- Use when a project needs its technical baseline or engine wiring clarified.

## Primary lane

- `control-plane`
- `architecture`
- `docs-release`

## Inputs

- Engine or stack choice
- Repo layout
- Setup constraints

## Steps

1. Identify the runtime or engine baseline.
2. List the required setup steps.
3. Call out missing configuration or risky assumptions.
4. Hand back a setup checklist.

## Outputs

- Baseline summary
- Setup steps
- Risks

## Exit criteria

- The project setup path is clear and actionable.

## Template

- `templates/setup-engine-plan.md`
