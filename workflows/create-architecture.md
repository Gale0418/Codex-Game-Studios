# Create Architecture

## Purpose

- Produce an architecture plan for a broad feature or project slice.

## Trigger

- Use when the technical shape needs to be written down before coding.

## Primary lane

- `architecture`
- `technical-director`
- `integration`

## Inputs

- Scope
- Constraints
- Existing code shape

## Steps

1. Identify the major components.
2. Define the ownership and boundaries.
3. Note the key risks and tradeoffs.
4. Hand back a concise architecture plan.

## Outputs

- Architecture
- Decisions
- Risks

## Exit criteria

- The implementation path is structurally clear.

## Template

- `templates/create-architecture-plan.md`
