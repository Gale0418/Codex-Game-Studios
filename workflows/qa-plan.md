# QA Plan

## Purpose

- Prepare the verification plan for a change or release.

## Trigger

- Use when tests, regressions, or release gating need to be planned.

## Primary lane

- `qa-lead`
- `qa-regression`
- `docs-release`

## Inputs

- Change scope
- Risk notes
- Acceptance criteria

## Steps

1. List the important behaviors to verify.
2. Separate smoke, regression, and gate checks.
3. Order the tests from fastest to slowest.
4. Hand back the QA plan.

## Outputs

- Test matrix
- Gate criteria
- Execution order

## Exit criteria

- The verification path is explicit and practical.

## Template

- `templates/qa-plan.md`
