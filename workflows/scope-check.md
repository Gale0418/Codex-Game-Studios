# Scope Check

## Purpose

- Check whether a task is too broad, too risky, or missing a boundary.

## Trigger

- Use when the request needs a scope decision.

## Primary lane

- `control-plane`
- `architecture`
- `producer`

## Inputs

- Request scope
- Constraints
- Risk notes

## Steps

1. Restate the scope in plain terms.
2. Flag what is in and out.
3. Name the biggest risk.
4. Hand back the recommendation.

## Outputs

- Scope
- Risks
- Recommendation

## Exit criteria

- The task boundary is explicit.

## Template

- `templates/scope-check.md`
