# Team Live Ops

## Purpose

- Run live-ops updates, events, and operational content work.

## Trigger

- Use when ongoing live content or operational cadence changes.

## Primary lane

- `live-ops-designer`
- `producer`
- `qa-regression`

## Inputs

- Live ops goals
- Content changes
- Risk notes

## Steps

1. Define the live ops change.
2. Note player-facing risk and timing.
3. Keep the operational scope small and clear.
4. Hand back the live ops plan.

## Outputs

- Live ops goals
- Changes
- Verification

## Exit criteria

- The live ops change is ready to run or clearly blocked.

## Template

- `templates/team-live-ops-plan.md`
