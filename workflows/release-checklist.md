# Release Checklist

## Purpose

- Confirm release readiness with a focused checklist.

## Trigger

- Use when the change is near shipping.

## Primary lane

- `release-manager`
- `qa-lead`
- `producer`

## Inputs

- Final diff
- QA evidence
- Release notes draft

## Steps

1. Check core readiness items.
2. Confirm blockers and known issues.
3. Keep the release summary concise.
4. Hand back the checklist result.

## Outputs

- Ready state
- Blockers
- Notes

## Exit criteria

- The release can proceed or is clearly blocked.

## Template

- `templates/release-checklist.md`
