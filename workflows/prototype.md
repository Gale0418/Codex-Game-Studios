# Prototype

## Purpose

- Build a lightweight prototype to test a hypothesis.

## Trigger

- Use when an idea needs fast validation.

## Primary lane

- `prototyper`
- `game-designer`
- `implementation`

## Inputs

- Hypothesis
- Constraints
- Validation check

## Steps

1. State the hypothesis.
2. Build the smallest useful prototype.
3. Check whether the idea holds.
4. Hand back the validation result.

## Outputs

- Hypothesis
- Prototype
- Validation

## Exit criteria

- The idea is proven enough to continue or reject.

## Template

- `templates/prototype.md`
