# Sprint Status

## Purpose

- Report the current sprint state, blockers, and next move.

## Trigger

- Use when the sprint needs a status update.

## Primary lane

- `producer`
- `architecture`
- `implementation`

## Inputs

- Sprint scope
- Current work
- Blockers

## Steps

1. State what is in progress.
2. Call out blockers or drift.
3. Name the next step.
4. Hand back the sprint status.

## Outputs

- Status
- Blockers
- Next steps

## Exit criteria

- The sprint state is clear enough to act on.

## Template

- `templates/sprint-status.md`
