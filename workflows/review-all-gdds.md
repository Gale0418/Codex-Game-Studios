# Review All GDDs

## Purpose

- Review the full design document set for consistency and gaps.

## Trigger

- Use when multiple design docs need a cross-check.

## Primary lane

- `creative-director`
- `game-designer`
- `docs-release`

## Inputs

- Design docs
- Source of truth
- Change list

## Steps

1. Scan the design set for conflicts.
2. Note missing or stale decisions.
3. Highlight the highest-value correction.
4. Hand back the review summary.

## Outputs

- Reviews
- Gaps
- Recommendation

## Exit criteria

- The design set is consistent enough to use.

## Template

- `templates/review-all-gdds.md`
