# Adopt

## Purpose

- Bring an existing repo or project into the studio flow safely.

## Trigger

- Use when you need to inherit an unfamiliar codebase or partially documented project.

## Primary lane

- `intake-scan`
- `control-plane`
- `docs-release`

## Inputs

- Existing repo structure
- Docs
- Current state

## Steps

1. Inspect what already exists.
2. Identify what is missing or stale.
3. Mark risky assumptions and ownership gaps.
4. Hand back an adoption checklist.

## Outputs

- Adoption notes
- Missing context
- Next steps

## Exit criteria

- The repo can enter the normal studio flow.

## Template

- `templates/adopt-checklist.md`
