# Bug Report

## Purpose

- Capture a bug report in a clear, reproducible format.

## Trigger

- Use when a bug needs to be written up or normalized.

## Primary lane

- `intake-scan`
- `qa-regression`
- `docs-release`

## Inputs

- Bug details
- Repro steps
- Expected vs actual

## Steps

1. Restate the problem.
2. Write the reproduction path.
3. Note expected and actual behavior.
4. Hand back the report.

## Outputs

- Repro steps
- Expected
- Actual

## Exit criteria

- Another person can reproduce the issue from the report.

## Template

- `templates/bug-report.md`
