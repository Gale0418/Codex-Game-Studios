# Localize

## Purpose

- Prepare strings or content for localization.

## Trigger

- Use when text needs to support multiple languages.

## Primary lane

- `localization-lead`
- `writer`
- `ui-ux`

## Inputs

- Strings
- Target languages
- UI constraints

## Steps

1. List the strings that need localization.
2. Note language or length constraints.
3. Mark any UI impacts.
4. Hand back the localization notes.

## Outputs

- Strings
- Languages
- Notes

## Exit criteria

- The content is ready for localization or clearly blocked.

## Template

- `templates/localize.md`
