# Test Helpers

## Purpose

- Define reusable test helpers or fixtures.

## Trigger

- Use when tests need shared setup or helpers.

## Primary lane

- `qa-regression`
- `integration`
- `docs-release`

## Inputs

- Helper request
- Test shape
- Constraints

## Steps

1. Identify the reusable behavior.
2. Define the helper shape.
3. Note usage and limitations.
4. Hand back the helper note.

## Outputs

- Helpers
- Usage
- Notes

## Exit criteria

- The helper is concrete enough to implement.

## Template

- `templates/test-helpers.md`
