# Tech Debt

## Purpose

- Review technical debt and decide what deserves attention now.

## Trigger

- Use when the codebase has accumulated maintainability risk.

## Primary lane

- `architecture`
- `producer`
- `docs-release`

## Inputs

- Debt list
- Risk notes
- Constraints

## Steps

1. Group debt by impact.
2. Flag what is worth paying down now.
3. Note what can safely wait.
4. Hand back the debt summary.

## Outputs

- Debt items
- Risks
- Notes

## Exit criteria

- The debt backlog is prioritized.

## Template

- `templates/tech-debt.md`
