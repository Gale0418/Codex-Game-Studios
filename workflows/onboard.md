# Onboard

## Purpose

- Bring a person, repo, or project into the studio flow.

## Trigger

- Use when onboarding context needs to be gathered quickly.

## Primary lane

- `control-plane`
- `intake-scan`
- `docs-release`

## Inputs

- Context
- Gaps
- Current state

## Steps

1. Gather the basic context.
2. Note what is missing.
3. Name the next safe step.
4. Hand back the onboarding summary.

## Outputs

- Context
- Gaps
- Next steps
- Wave goal if the onboarding step hands off to subagents

## Exit criteria

- The onboarding path is clear.

## Template

- `templates/onboard.md`
