# Dev Story

## Purpose

- Execute one story with a small, reviewable patch.

## Trigger

- Use when a story is ready for implementation.

## Primary lane

- `implementation`
- `integration`
- `qa-regression`

## Inputs

- Story
- Acceptance notes
- Relevant files

## Steps

1. Keep the patch small and coherent.
2. Update only the files needed for the story.
3. Verify the change with the smallest useful check.
4. Hand back the patch plan and verification status.

## Outputs

- Patch plan
- Verification
- Handoff

## Exit criteria

- The story is implemented or clearly blocked.

## Template

- `templates/dev-story-plan.md`
