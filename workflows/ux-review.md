# UX Review

## Purpose

- Review a UI or flow for usability, clarity, and consistency.

## Trigger

- Use when a UI change needs a review before shipping.

## Primary lane

- `ui-ux`
- `qa-regression`
- `docs-release`

## Inputs

- UI diff
- User flow
- Acceptance criteria

## Steps

1. Inspect the interaction flow.
2. Note usability concerns and inconsistencies.
3. Recommend the cleanest fix or follow-up.
4. Hand back the review summary.

## Outputs

- Findings
- Concerns
- Recommendation

## Exit criteria

- The UI decision is actionable.

## Template

- `templates/ux-review.md`
