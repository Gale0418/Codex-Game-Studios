# Story Done

## Purpose

- Confirm a story has met its done criteria.

## Trigger

- Use when a story should be closed.

## Primary lane

- `qa-regression`
- `integration`
- `docs-release`

## Inputs

- Story
- Done criteria
- Verification notes

## Steps

1. Check the done criteria.
2. Confirm the supporting verification.
3. Note anything that still needs follow-up.
4. Hand back the closeout note.

## Outputs

- Story
- Done criteria
- Notes

## Exit criteria

- The story can be marked done or is clearly not done.

## Template

- `templates/story-done.md`
