# Create Control Manifest

## Purpose

- Write down the routing and control-plane decisions for the studio flow.

## Trigger

- Use when a task needs an explicit routing manifest.

## Primary lane

- `control-plane`
- `workflow-catalog`
- `technical-director`

## Inputs

- Commands
- Workflows
- Routing rules

## Steps

1. List the command-to-workflow routes.
2. Note the control-plane rules.
3. Call out any ambiguous routing.
4. Hand back the manifest note.

## Outputs

- Controls
- Routes
- Notes

## Exit criteria

- The studio routing can be read as a single source of truth.

## Template

- `templates/create-control-manifest.md`
