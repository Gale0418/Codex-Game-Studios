# Retrospective

## Purpose

- Capture lessons, wins, and actions after a work cycle.

## Trigger

- Use after a sprint, release, or major effort.

## Primary lane

- `producer`
- `qa-lead`
- `docs-release`

## Inputs

- Completed work
- Issues
- Lessons learned

## Steps

1. Note what went well.
2. Note what should improve.
3. Name the concrete action items.
4. Hand back the retrospective.

## Outputs

- Went well
- Improve
- Actions

## Exit criteria

- The team leaves with concrete follow-up work.

## Template

- `templates/retrospective.md`
