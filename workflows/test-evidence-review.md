# Test Evidence Review

## Purpose

- Review the evidence produced by a test pass.

## Trigger

- Use when test output needs interpretation or gate review.

## Primary lane

- `qa-lead`
- `qa-regression`
- `docs-release`

## Inputs

- Evidence
- Expected outcomes
- Gate criteria

## Steps

1. Read the evidence.
2. Compare it to the gate criteria.
3. Note any weak spots or missing proof.
4. Hand back the review.

## Outputs

- Evidence
- Review
- Recommendation

## Exit criteria

- The evidence is enough to support a go / no-go call.

- `templates/test-evidence-review.md`
