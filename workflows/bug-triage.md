# Bug Triage

## Purpose

- Prioritize and route a bug after it is reported.

## Trigger

- Use when a bug needs triage and ownership.

## Primary lane

- `qa-regression`
- `producer`
- `docs-release`

## Inputs

- Bug report
- Impact
- Repro confidence

## Steps

1. Assess priority.
2. Assign the likely owner.
3. Note the next action.
4. Hand back the triage note.

## Outputs

- Bug
- Priority
- Next step

## Exit criteria

- The bug has an owner and a priority.

## Template

- `templates/bug-triage.md`
