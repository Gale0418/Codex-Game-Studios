# Asset Audit

## Purpose

- Review assets for gaps, stale references, and production risk.

## Trigger

- Use when assets need a consistency or readiness pass.

## Primary lane

- `art-director`
- `technical-artist`
- `docs-release`

## Inputs

- Asset set
- Style guide
- Usage targets

## Steps

1. Inspect the asset surface.
2. Find missing or inconsistent assets.
3. Call out the highest-risk gaps.
4. Hand back the audit summary.

## Outputs

- Audit scope
- Gaps
- Action items

## Exit criteria

- The asset set is ready to use or clearly blocked.

## Template

- `templates/asset-audit.md`
