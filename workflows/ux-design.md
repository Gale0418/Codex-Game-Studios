# UX Design

## Purpose

- Design interaction and layout behavior before implementation.

## Trigger

- Use when UI behavior or user flow needs a design pass.

## Primary lane

- `ui-ux`
- `creative-director`
- `implementation`

## Inputs

- Interaction goals
- Layout constraints
- Target devices

## Steps

1. State the interaction goal.
2. Define layout and flow choices.
3. Call out the biggest UX risks.
4. Hand back the UX design notes.

## Outputs

- Interaction goals
- Layout notes
- Risks

## Exit criteria

- The UI can move into implementation with clear intent.

## Template

- `templates/ux-design.md`
