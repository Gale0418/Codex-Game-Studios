# Starter Tree

```text
game-project/
  README.md
  src/
    .gitkeep
  assets/
    .gitkeep
  tests/
    .gitkeep
  design/
    .gitkeep
  docs/
    .gitkeep
```

## Why this is enough

- The root `README.md` explains the project and the intended stack.
- `src/` exists before code lands, so ownership is obvious.
- `assets/` exists before art lands, so imports stay organized.
- `tests/` exists before the first bug hunt, so verification has a home.
- `design/` exists before the scope grows, so decisions do not get lost.
- `docs/` exists before handoff, so onboarding does not depend on memory.
