# Game Project Skeleton Example

This folder shows the smallest practical layout for a new game project.
It is engine-agnostic on purpose, so the studio can point it at Godot, Unity, Unreal, or a custom stack.

Use this as the starting point:

- `src/` for game logic and runtime code.
- `assets/` for art, audio, and imported media.
- `tests/` for smoke and regression coverage.
- `design/` for planning, balance, and layout notes.
- `docs/` for setup, build, and release instructions.

If the project grows, add `content/` for narrative or quest data and `tools/` for import or build helpers.
